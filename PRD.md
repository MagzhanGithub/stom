## 1. Project Overview

**Проект:** Сайт стоматологической клиники Zhanar Dent

**Клиника:** Zhanar Dent
**Адрес:** г. Шымкент, ул. Байтерекова 9/3
**Рынок:** Казахстан, город Шымкент (русскоязычная аудитория)

**Описание:** Профессиональный, конверсионно-ориентированный веб-сайт для стоматологической клиники. Цель — стать главным каналом привлечения новых пациентов, формирования доверия и онлайн-записи.

**Проблема, которую решает сайт:**
- Пациенты ищут стоматолога онлайн, но существующие сайты конкурентов медленные, неудобные и не вызывают доверия
- 67% пациентов предпочитают онлайн-запись, но большинство клиник предлагают только звонок
- 36% людей испытывают стоматологическую тревогу — сайт должен её снижать, а не усиливать

**Видение:** Современный, быстрый, вызывающий доверие сайт, который работает как лучший "продавец" клиники 24/7 и конвертирует посетителей в записанных пациентов.

**Бренд-цвет:** `#4ddde2` (teal)

## 3. Target Audience

### Первичная персона: "Беспокоящийся пациент"
- **Возраст:** 28–55 лет
- **Пол:** Преимущественно женщины (принимают решения о здоровье семьи)
- **Устройство:** Мобильный телефон (73% запросов)
- **Боли:**
  - Боится боли у стоматолога
  - Не знает, сколько стоит лечение
  - Не доверяет незнакомым врачам
  - Сложно дозвониться в клинику
- **Мотивации:**
  - Хочет убедиться, что клиника безопасна и современна
  - Ищет реальные отзывы и фото "до/после"
  - Хочет записаться быстро, без звонка, в удобное время
- **Ключевые страхи:** Боль, высокая цена, некомпетентный врач

### Вторичная персона: "Мама, ищущая для ребёнка"
- **Возраст:** 25–40 лет
- **Боли:** Ищет врача, который умеет работать с детьми, в комфортной обстановке
- **Мотивации:** Безопасность, доброжелательность персонала, удобное расписание

### Третичная персона: "Пациент на имплантацию / косметику"
- **Возраст:** 35–60 лет
- **Боли:** Неуверенность в результате, страх высокой цены
- **Мотивации:** Галерея "до/после", портфолио конкретного врача, прозрачное ценообразование

| Роль | Цвет | HEX | Применение |

| Primary (Brand) | Teal | `#4ddde2` | Заголовки-акценты, иконки, ссылки |

| Primary Light | Light Teal | `#b2f4f6` | Фоны секций, tints |
| CTA | Orange | `#FF6B35` | Кнопки записи (главный CTA) |
| CTA Hover | Dark Orange | `#E55A24` | Hover для CTA кнопок |
| Dark | Navy | `#0d1a2b` | Навигация, footer |
| Dark 2 | Dark Navy | `#162132` | Footer background |
| Text | Slate | `#1e2e3f` | Основной текст |
| Muted | Gray | `#6b7c93` | Вторичный текст, подписи |

**Правило 1:** Красный цвет — ЗАПРЕЩЁН в брендинге и CTA. Только для валидации форм.
**Правило 2:** Белый текст на teal (#4ddde2) — ЗАПРЕЩЁН. Контраст 2.1:1 не проходит WCAG. Teal — только акцент/иконки/borders.
**Правило 3:** Полная token-система определена в [DESIGN.md — Section 2](DESIGN.md).

### 7.6 Компоненты

**Кнопки:**
- Primary CTA: оранжевый фон, белый текст, border-radius 9999px, min-height 52px, min-width 180px
- Secondary: белый фон, teal border, teal текст
- Ghost: прозрачный, белый текст (только на тёмных фонах)
- Hover: scale(1.02) + shadow elevation

**Карточки услуг:**
- Белый фон, shadow-sm, border-radius 12px
- Иконка (teal) + заголовок + описание + "Подробнее" ссылка
- Hover: shadow-md + translateY(-4px)

**Навигация:**
- Desktop: sticky, белый фон + blur, логотип слева, ссылки по центру, [тел + Записаться] справа
- Mobile: hamburger справа, sticky bottom bar с телефоном и CTA

### 7.7 Анимации

```
transition: 300ms cubic-bezier(.4, 0, .2, 1)
Scroll-triggered: fadeUp (opacity 0→1, translateY 24px→0), delay по очереди
Number counters: countUp при первом появлении в viewport
Hover cards: translateY(-4px), 250ms
Hero video: loop, muted, autoplay, poster fallback
Drag-to-reveal slider: плавный, с иконкой-указателем
```

**Правило анимаций:** Максимум 300ms, subtle. Стоматофобия — никаких резких, дёргающихся элементов.

## 10. Technical Requirements

### 10.1 Performance

| Метрика | Цель |
|---------|------|
| LCP | < 2.5s |
| INP | < 200ms |
| CLS | < 0.1 |
| Google PageSpeed (mobile) | ≥ 90 |
| Time to First Byte (TTFB) | < 800ms |
| Размер страницы (homepage) | < 1.5MB |
| Hero image | < 200KB (WebP) |
| Video hero | < 8MB (WebM + MP4) |

**Оптимизации:**
- Все изображения в WebP + AVIF
- Lazy loading для изображений ниже fold
- `srcset` для responsive images
- Critical CSS inline, остальное async
- Google Fonts через `preconnect` + display=swap
- CDN для статики (Cloudflare или аналог)

### 10.2 SEO & Schema

**Обязательные Schema Types (JSON-LD):**
```
LocalBusiness / Dentist
├── name, address, telephone, geo
├── openingHours, priceRange
├── aggregateRating (из Google Reviews)
├── availableService (для каждой услуги)
└── hasMap

FAQPage — на главной + каждой странице услуги
BreadcrumbList — на всех внутренних страницах
Person — для каждого врача
MedicalProcedure — для каждой процедуры (P2)
```

**Meta-теги:**
- Уникальный `<title>` для каждой страницы (55–60 символов)
- Уникальный `<meta description>` (150–160 символов)
- `hreflang` если будет несколько языков
- Open Graph для социальных сетей
- `<link rel="canonical">` на каждой странице

**robots.txt + XML sitemap** → Google Search Console

### 10.3 Доступность (WCAG 2.1 AA)

- Контраст текст/фон: минимум 4.5:1
- Все кнопки и ссылки с `aria-label`
- `alt` текст для всех изображений
- Keyboard navigation (Tab order логичный)
- Skip navigation link
- Focus visible indicators
- Видео-субтитры (если есть видео с речью)

### 10.7 Tech Stack (рекомендация)

**✅ РЕШЕНИЕ: Next.js 14 (App Router) + TypeScript + Tailwind CSS**

| Технология | Роль | Почему |
|-----------|------|--------|
| Next.js 14 (App Router) | Фреймворк | SSG/ISR — максимальная скорость, SEO из коробки, файловый роутинг |
| TypeScript | Язык | Надёжность, автодополнение, меньше багов при правках |
| Tailwind CSS v3 | Стили | Правки дизайна в одну строку, JIT, нет лишнего CSS |
| Framer Motion | Анимации | Плавные scroll-triggered анимации, drag-to-reveal слайдер |
| next/image | Изображения | Автоматическая оптимизация, WebP, lazy loading |
| next/font | Шрифты | Google Fonts без layout shift |

**Структура проекта:**
```
/app              — страницы (App Router)
/components       — переиспользуемые компоненты
  /ui             — атомарные (Button, Card, Badge)
  /sections       — секции страниц (Hero, Services, FAQ...)
  /layout         — Header, Footer, Navigation
/lib              — утилиты, константы, данные
/public           — статичные файлы, изображения
/styles           — globals.css (Tailwind directives)
```

**Почему легко адаптировать:**
- Клиент хочет новую секцию → добавить компонент в `/components/sections/`
- Клиент хочет изменить цвет → поменять одну переменную в `tailwind.config.ts`
- Клиент хочет новую страницу → создать файл в `/app/`
- Клиент хочет обновить цены/услуги → редактировать данные в `/lib/data.ts`
